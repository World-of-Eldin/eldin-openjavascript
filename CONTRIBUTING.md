# Contributing to OpenJS

Thank you for considering contributing to OpenJS.

Whether you’re fixing a bug, improving documentation, or adding new features, contributions are welcome. Follow the steps below to get started.

---

## Reporting Issues

If you find a bug or unexpected behavior, please open an issue with:

- A clear title and description
- Steps to reproduce the issue
- Your Minecraft version and OpenJS version
- Relevant logs or script snippets if possible

---

## Submitting Code Changes

1. Fork the repository
2. Create a new branch with a descriptive name  
   (e.g. `fix-nullpointer-scriptloader`)
3. Make your changes
4. Test your code with the latest Folia and Paper builds
5. Submit a pull request (PR)

Please keep PRs focused and avoid bundling unrelated changes.

---

## Coding Guidelines

- Use consistent formatting (Java: 4-space indentation, no tabs)
- Keep scripting API simple and minimal
- Avoid hard dependencies on third-party plugins unless optional
- New features must not break existing scripts
- Retain Folia and Paper support

---

## Development Setup

You’ll need:

- Java 17+
- Maven
- A test server (Folia or Paper) on v1.13+
- A debugger or console log setup

---
